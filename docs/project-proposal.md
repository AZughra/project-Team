مشروع تطبيق متجر إلكتروني
1. المعلومات الأساسية والوصف
*/اسم المشروع (SmartShop)
منصة متجر إلكتروني بسيطة وسهلة الاستخدام.
*/ نظرة عامة
هو تطبيق متجر إلكتروني يتيح للمستخدمين تصفح المنتجات، إضافتها إلى سلة المشتريات، وإتمام عملية الشراء عبر الإنترنت بسهولة وأمان. يهدف المشروع إلى توفير تجربة تسوق رقمية سلسة لكل من العملاء وإدارة المتجر.
*/بيان المشكلة
العديد من المتاجر الصغيرة والمتوسطة تفتقر إلى حلول رقمية سهلة وسريعة لعرض منتجاتها وبيعها عبر الإنترنت. كما يواجه العملاء صعوبة في استخدام بعض المتاجر الإلكترونية المعقدة، مما يقلل من رغبتهم في الشراء. هذا التطبيق يأتي لتقديم حل بسيط وفعّال للطرفين. 2/ تحديد المستخدمين
   1) العميل (Customer)
    الوصف: شخص يرغب في شراء منتجات عبر الإنترنت باستخدام الهاتف أو المتصفح.
    لأهداف: تصفح المنتجات بسهولة، معرفة الأسعار، إتمام الشراء بسرعة وأمان.
    نقاط الألم: تعقيد واجهات بعض المتاجر، بطء عملية الشراء، وعدم وضوح معلومات المنتجات.
   2) مدير المتجر (Store Admin)
    الوصف: صاحب المتجر أو المسؤول عن إدارة المنتجات والطلبات.
    الأهداف: إضافة وتعديل المنتجات، متابعة الطلبات، إدارة المخزون.
    نقاط الألم: صعوبة إدارة الطلبات يدوياً، وعدم وجود تقارير واضحة عن المبيعات.
   3) مدير النظام (System Admin)
    الوصف: الشخص المسؤول عن تشغيل النظام وصيانته.
    الأهداف: ضمان استقرار النظام، إدارة المستخدمين، وحل المشاكل التقنية.
    نقاط الألم: الأعطال المفاجئة وصعوبة تتبع الأخطاء دون أدوات مناسبة

3/ الميزات الأساسية والإضافية
*/الميزات الجوهرية
1/تسجيل الدخول وإنشاء حساب: مكين المستخدمين من إنشاء حسابات شخصية (يسهّل عملية الشراء والمتابعة)
2/ تصفح المنتجات: عرض المنتجات مع الصور والأسعار والوصف (يساعد العميل على اتخاذ قرار الشراء).
3/ سلة المشتريات: إضافة وحذف المنتجات قبل الشراء (مرونة للمستخدم)
4/ إتمام الطلب (Checkout):إدخال بيانات الشحن وتأكيد الطلب (الهدف الأساسي للتطبيق).
5/ لوحة تحكم مدير المتجر:إدارة المنتجات والطلبات (تنظيم العمل وزيادة الكفاءة)
*/ الميزات الإضافية
* نظام تقييم ومراجعات للمنتجات.
* البحث المتقدم وتصفية المنتجات.
* دعم كوبونات الخصم.
* إشعارات للطلبات

4/ الجانب التقني والجدول الزمني
*/ مجموعة التقنيات
Mobile App: Android Studio
Language: java
UI: Xml 
Database: Sqlite Database
Tools: Git , GitHub , Android Emulator , Figma
*/ هيكلية النظام
المستخدم <-- تطبيق الموبايل <-- قاعدة البيانات السحابية
*/ الجدول الزمني للمشروع 
الاسبوع 3 - 4 تصميم الواجهات و تحليل المتطلبات 
الاسبوع 5-6 قاعدة البيانات 
الاسبوع 7 - 8 تطوير الوجهات الامامية
الاسبوع 9 - 10 تطوير المشروع كامل 
الاسبوع 11 - 12 اختبار التطبيق و اصلاح الاخطاء 
الاسبوع 13 - 14 تجهيز عرص المشروع بعد تحسينه 

5/ معايير النجاح والمخاطر
*/ معايير النجاح
* تنفيذ جميع الميزات الأساسية (MVP).
* تغطية اختبارات برمجية لا تقل عن 70%.
* واجهة سهلة الاستخدام وتجربة مستخدم جيدة.
* استقرار النظام دون أخطاء حرجة.

* المخاطر والحلول
1/ تأخر أحد أعضاء الفريق:توزيع المهام بشكل مرن وتوثيق العمل باستمرار.
2/ صعوبة تقنية غير متوقعة:البحث المبكر وطلب الدعم أو تبسيط الميزة.
3/ ضيق الوقت:التركيز على الميزات الجوهرية وتأجيل الإضافية.

6/ الأدوار والمسؤوليات
*/ Product Owner
* تحديد متطلبات المشروع والأولويات.
* التواصل مع الفريق ومتابعة تحقيق الأهداف.
*/ Scrum Master
* تنظيم الـ Sprints والاجتماعات.
* إزالة العوائق التي تواجه الفريق.
*/ Lead Developer
* الإشراف على الجانب التقني.
* اتخاذ القرارات المعمارية ومراجعة الكود.
