# مقترح مشروع: سلتي (Sallaty)
## 1. المعلومات الأساسية والوصف

### اسم المشروع
Sallaty - منصة متجر إلكتروني بسيطة وسهلة الاستخدام.

### نظرة عامة (Overview)
تطبيق متجر إلكتروني يتيح للمستخدمين تصفح المنتجات، إضافتها إلى سلة المشتريات، وإتمام عملية الشراء عبر الإنترنت بسهولة وأمان. يهدف المشروع إلى توفير تجربة تسوق رقمية سلسة لكل من العملاء وإدارة المتجر.

### بيان المشكلة (Problem Statement)
* تفتقر المتاجر الصغيرة والمتوسطة إلى حلول رقمية سهلة وسريعة لعرض وبيع منتجاتها.
* يواجه العملاء صعوبة في التعامل مع الواجهات المعقدة، مما يقلل من رغبتهم في الشراء.
* الحل: يقدم التطبيق واجهة بسيطة وفعّالة تخدم الطرفين وتسهل عملية التحول الرقمي.

---

## 2. تحديد المستخدمين (User Personas)

### العميل (Customer)
* الوصف: شخص يرغب في شراء منتجات عبر الإنترنت باستخدام الهاتف أو المتصفح.
* الأهداف: تصفح المنتجات بسهولة، معرفة الأسعار، وإتمام الشراء بسرعة وأمان.
* نقاط الألم: تعقيد الواجهات، بطء عملية الشراء، وعدم وضوح معلومات المنتجات.

### مدير المتجر (Store Admin)
* الوصف: صاحب المتجر أو المسؤول عن إدارة المنتجات والطلبات.
* الأهداف: إضافة وتعديل المنتجات، متابعة الطلبات، وإدارة المخزون.
* نقاط الألم: صعوبة إدارة الطلبات يدوياً، وغياب تقارير المبيعات الواضحة.

### مدير النظام (System Admin)
* الوصف: المسؤول عن تشغيل النظام وصيانته.
* الأهداف: ضمان استقرار النظام، إدارة المستخدمين، وحل المشاكل التقنية.
* نقاط الألم: الأعطال المفاجئة وصعوبة تتبع الأخطاء البرمجية.

---

## 3. الميزات الأساسية والإضافية

### الميزات الجوهرية (Core Features)
1. تسجيل الدخول وإنشاء حساب: تمكين المستخدمين من إنشاء حسابات شخصية لتسهيل الشراء والمتابعة.
2. تصفح المنتجات: عرض المنتجات مع الصور والأسعار والوصف لمساعدة العميل على اتخاذ القرار.
3. سلة المشتريات: مرونة كاملة في إضافة وحذف المنتجات قبل الشراء.
4. إتمام الطلب (Checkout): إدخال بيانات الشحن وتأكيد الطلب.
5. لوحة تحكم مدير المتجر: تنظيم العمل وزيادة الكفاءة عبر إدارة المنتجات والطلبات.

### الميزات الإضافية (Bonus Features)
* نظام تقييم ومراجعات للمنتجات.
* البحث المتقدم وتصفية المنتجات.
* دعم كوبونات الخصم.
* إشعارات للطلبات.

## 4. الجانب التقني والجدول الزمني

### مجموعة التقنيات (Tech Stack)
* تطبيق الموبايل: Android Studio
* اللغة البرمجية: Java
* واجهة المستخدم: XML
* قاعدة البيانات: SQLite Database
* الأدوات المساندة: Git, GitHub, Android Emulator, Figma

### هيكلية النظام (Architecture)
المستخدم <--- تطبيق الموبايل <--- قاعدة البيانات

### الجدول الزمني للمشروع (Sprint Plan)
* الأسبوع 3 - 4: Sprint 0 (التخطيط وتحليل المتطلبات وتصميم الواجهات).
* الأسبوع 5 - 6: Sprint 1 (بناء قاعدة بيانات SQLite ونظام تسجيل الدخول).
* الأسبوع 7 - 8: Sprint 2 (تطوير واجهة عرض المنتجات وسلة المشتريات).
* الأسبوع 9 - 10: Sprint 3 (لوحة تحكم مدير المتجر والربط الكامل).
* الأسبوع 11 - 12: Sprint 4 (اختبار التطبيق وإصلاح الأخطاء).
* الأسبوع 13 - 14: Sprint 5 (تجهيز العرض التقديمي والتقرير الفني).

## 5. معايير النجاح والمخاطر

### معايير النجاح
* تنفيذ جميع الميزات الأساسية (MVP).
* تغطية اختبارات برمجية لا تقل عن 70%.
* واجهة مستخدم سهلة واستقرار النظام.

### المخاطر والحلول
1. تأخر أعضاء الفريق: الحل هو توزيع المهام بمرونة وتوثيق العمل دورياً.
2. صعوبات تقنية: الحل هو البحث المبكر وتبسيط الميزات المعقدة.
3. ضيق الوقت: الحل هو التركيز على الميزات الجوهرية وتأجيل الإضافية.

## 6. الأدوار والمسؤوليات (Scrum Roles)

* Product Owner (تامر مازن زكريا الجياز): تحديد الأولويات وضمان تلبية احتياجات المستخدمين.
* Scrum Master (أسامة محمد ابو جلمبو): إدارة تدفق العمل، تنظيم الاجتماعات، وإزالة العوائق.
* Development Team (احمد عماد زغرة ومطر سليمان طوطح): بناء التطبيق، تصميم القواعد، وتحويل المتطلبات إلى كود.
